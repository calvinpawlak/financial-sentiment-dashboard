## What this project is

A personal, free-data stock sentiment dashboard (owner: Calvin,
calvinplax@icloud.com). Full description in `PROJECT_CONTEXT.md`. One-line
version: ingest prices + public sentiment for a fixed 9-ticker list, score
it, compute a conservative BUY/SELL/HOLD Signal, track whether the Signal
was actually right, show all of it on a dashboard. Not financial advice.

## Decision rules

- **Free-tier / no-paid-services is the default assumption.** If a
  problem seems to call for a paid API tier or paid hosting, say so
  explicitly and ask rather than assuming Calvin wants to pay.
- **Prefer a free workaround over a paid replacement** when a source
  breaks or gets rate-limited or changes terms.
- **Keep the BUY/SELL/HOLD rule conservative** — HOLD whenever sentiment
  and price direction disagree or data is insufficient. Don't make it more
  decisive without an explicit request to change that trade-off.
- **Don't silently change grading/statistical methodology** — the
  accuracy log's 4h/24h horizons, `MIN_GRADED_FOR_CONFIDENCE = 20`, and
  the Wilson confidence interval choice were deliberate. Propose and
  explain before changing them.
- **When in doubt about scope, ask rather than assume** — this project's
  history shows a preference for being asked about hosting/architecture
  tradeoffs rather than having a default silently chosen.

## Running this project

- Preferred interpreter: `.\.venv\Scripts\python.exe` (Python 3.12,
  matching Render). `setup_task_scheduler.ps1` uses it automatically when
  present.

- `python main.py` — one full ingestion cycle (all sources).
- `python main.py --fast-only` — prices/StockTwits/Reddit/Finnhub only
  (matches the 5-minute scheduled job).
- `python main.py --slow-only` — FinViz/Google News only (matches the
  15-minute scheduled job).
- `python report.py` — quick CLI sentiment/signal summary.
- `python webapp/server.py` — local Flask dashboard at
  `http://localhost:5000`.
- Full command reference and setup steps: `README.md`. Recurring
  processes with trigger/steps/completion-criteria: `WORKFLOWS.md`.

## Deployment

Public dashboard: Render (`financial-sentiment-dashboard.onrender.com`),
auto-deploys from GitHub (`calvinpawlak/financial-sentiment-dashboard`) on
push. Database: Neon (Postgres), connected via `DATABASE_URL`. Full detail
and how-to-test steps for each: `CONNECTORS.md`. Note:
`webapp/server.py` does not call `init_db()` — schema changes only reach
the hosted Neon database via a local `python main.py` run with
`DATABASE_URL` set (see `WORKFLOWS.md`, workflow 7).
